### 1.2 名词定义

| 术语 | 说明 |
| --- | --- |
| 画册（BabyBook / Book） | 由多张宝宝照片按模板排版生成的电子相册（MVP 阶段仅 Web 端本地生成） |
| 模板（Template） | 决定画册整体风格的版式方案，包括封面、内页、背景、字体、装饰元素、配色等 |
| 内置模板（Built-in Template） | 随版本发布、以代码常量形式内置的官方模板（当前 **30 款**），用户不可修改 |
| 用户模板 / 自定义模板（User Template） | 用户通过可视化编辑器自行创建并保存在本地 IndexedDB 中的模板，可随时编辑和删除 |
| 风格（TemplateStyle） | 模板大类，如水彩手绘、萌趣卡通、清新极简、复古胶片、中国红、圣诞节 |
| 页面（BookPage） | 画册中的单页，由图片位、文字位、版式类型（layout）组成 |
| 版式（PageLayoutType） | 单页图文的排列组合方式，如封面、单图、三宫格、四/五/六宫格等 |
| 骨架变体（Layout Variant） | 多图版式（double/triple/grid4/grid5/grid6）下"照片块位置/尺寸/旋转"的可切换方案；只影响骨架，不影响风格装饰（相框、气泡等仍由 `style` 决定） |
| 双页书本（Spread） | 一本画册在预览/详情页中以"对开书"形式呈现，左右两页 + 翻页动画 |
| 站点主题（SiteTheme） | 站点外壳（首页、Header、列表页）的视觉风格，与画册内页风格**完全解耦**，共 4 套预设 |
| 画册主题（BookTheme） | 每本画册的覆盖项（封面图、标题色、相框色等），与模板正交，保证同一模板在不同画册仍可个性化 |

## 二、项目背景与目标

### 2.1 背景

家长在宝宝成长过程中会拍摄海量照片（月龄照、节庆照、日常照），普遍存在三类痛点：

1. **整理难**：照片散落在相册、微信、云盘，缺乏结构化归档。
2. **呈现难**：专业排版门槛高；现成第三方相册 App 普遍风格单一、节庆/宝宝向模板稀缺。
3. **分享/留念难**：希望生成可分享的链接/图片，或打印成实体画册作为成长礼物。

### 2.2 产品定位

一款面向家长的「**AI 辅助排版 + 丰富成长主题模板**」画册工具。MVP 阶段以 **Web 端本地运行** 为起点（所有数据存浏览器 IndexedDB，不上传服务器），聚焦"挑模板 → 上传照片 → 一键生成 → 预览 → 本地导出"的主链路。

### 2.3 目标用户

- **核心**：0–6 岁宝宝家长（妈妈占主导，20–45 岁）
- **扩展**：祖父母希望收到打印画册；幼师制作班级成长册

### 2.4 产品目标（MVP v1.4）

| 目标 | 衡量指标 | 状态 |
| --- | --- | --- |
| 低门槛出片：非设计师 5 分钟内完成一本画册 | 创建耗时 ≤ 5min；向导步数 ≤ 4 | 已实现（4 步向导） |
| 丰富模板：至少覆盖 5 大风格 × 10 款起步 | 模板数 ≥ 10 | 已超额（**30 款 / 6 风格**，v1.4） |

## 四、详细需求

### 4.1 首页 `/` ✅

**目标**：让新用户在 5 秒内理解产品能做什么，并给出清晰的下一步。

**页面结构**（由 `src/pages/HomePage.tsx` 实现）：

1. Hero 区：主标题 + 副标题 + 两个 CTA
   - 主 CTA：「开始创建画册」→ `/create`
   - 次 CTA：「浏览模板库」→ `/templates`
2. 特色模板展示：横排展示 3–4 款精选模板（点击进入模板预览）
3. 三大价值点图文：智能排版 / 丰富模板 / 一键分享
4. 底部入口：「我的画册」→ `/my`

### 4.2 模板库 `/templates` ✅

**目标**：让用户在进入创建流程前/中快速找到合适的模板，并管理自己创建的自定义模板。

**功能**（由 `src/pages/TemplatesPage.tsx` + `src/components/TemplatePreviewModal.tsx` 实现）：

- **筛选标签**：
  - 按 `TemplateStyle` 六大风格过滤（水彩手绘 / 萌趣卡通 / 清新极简 / 复古胶片 / 中国红 / 圣诞）
  - 按**来源**过滤：全部 / 官方模板 / 我的模板（自定义）
- **搜索**：按模板名 / 描述关键词实时过滤
- **模板卡**：
  - 封面缩略图 + 名称 + 风格标签 + 简短描述
  - 自定义模板卡片右上角显示「我的」徽标
  - 卡片底部操作：**预览**（所有模板通用）/ **编辑** · **删除**（仅自定义模板）
  - 点击卡片主体打开预览
- **顶部操作**：右上角「+ 创建自定义模板」按钮 → `/templates/new`
- **预览弹层**：
  - 全屏弹出，展示该模板的**示例画册**（使用内置示例照片）
  - 以**双页书本对开**形式呈现，带翻页动画
  - 左右半区点击 / `←` `→` 键盘翻页 / 页码圆点跳转
  - 底部紧凑控件条：翻页按钮 + 页码圆点 + 页码文本
  - **关键体验**：BookStage 组件通过 `ResizeObserver` 实测可用宽高，按 3:2 双页比例夹紧，**保证任何窗口尺寸下都完整展示整本画册，不需要滚动**
  - 右上角「使用此模板」→ 跳转 `/create`（携带 `templateId`）

### 4.3 创建画册 `/create` ✅

**目标**：一条通畅主路径，引导用户 5 分钟内出片。

**流程**（由 `src/pages/CreatePage.tsx` 实现，共 4 步）：

#### 步骤 1：上传照片
- 支持多选上传（图片格式 jpg/png/webp）
- 上传后**立即在浏览器内压缩**（`src/imageUtils.ts`）：
  - 最长边 ≤ 1600px
  - JPEG 质量 0.85
  - 并发度 3（避免主线程卡顿）
- 缩略图网格展示，支持删除、调整顺序（拖拽）
- 最少 4 张，最多 60 张（超出提示）

#### 步骤 2：基本信息
- 画册标题（必填，默认「XX 的成长画册」）
- 宝宝昵称、日期范围（可选）
- 封面主标题、副标题（可选，默认由模板生成）

#### 步骤 3：选择模板
- 以卡片网格展示全部模板（内置 10 款 + 用户自定义模板）
- 点击卡片高亮选中，可点「预览」打开双页对开预览弹层
- 也支持从模板库 `/templates` 深链带入 `templateId`（含自定义模板 id）

#### 步骤 4：生成并预览
- 进度条动画（压缩 → 排版 → 渲染）
- 排版引擎（`src/layoutEngine.ts`）根据照片尺寸、数量智能选版式（详见 4.6）
- 完成后自动跳转 `/book/:id`

### 4.4 我的画册 `/my` ✅

**目标**：快速回到任意一本已创建的画册。

**功能**（由 `src/pages/MyBooksPage.tsx` 实现）：

- 卡片网格展示：封面缩略 + 标题 + 页数 + 创建时间（相对时间）
- 点击卡片 → 进入 `/book/:id`
- 右上角删除按钮（二次确认后从 IndexedDB 删除）
- 空态：插画 + 「创建第一本画册」CTA

### 4.5 画册详情 `/book/:id` ✅

**目标**：让画册以"真实书本"的形式被欣赏和分享。

**功能**（由 `src/pages/BookDetailPage.tsx` 实现）：

- **双页书本预览**：左页 + 右页对开，封面单页居中
- **翻页动画**：CSS 3D transform，约 600ms 缓动翻页
- **交互**：
  - 点击左右半区翻页
  - `←` / `→` 键盘翻页
  - 底部页码圆点跳转到任意对开
- **导出**：
  - 「导出当前页 PNG」：`html2canvas` 截取当前对开
  - 「导出长图 PNG」：全书所有页纵向拼接为一张长图
- **编辑**（轻量）：修改画册标题、切换模板（重新排版）

### 4.6 智能排版引擎 ✅

**目标**：让照片自动"落位"到最合适的版式，减少用户手动调整。

**规则**（由 `src/layoutEngine.ts` 实现）：

1. **首页**：始终为 `cover`（封面）
2. **末页**：始终为 `ending`（结束页）
3. **内页（根据剩余照片队列动态取用，优先匹配能吃掉更多照片的版式）**：
   - 若下一张照片 **ratio ≥ 1.25**（明显横图）→ `single`（整页单图横幅）
   - 若下一张照片 **ratio ≤ 0.8**（明显竖图）→ `single-portrait`（竖图专版，右侧留白放文字）
   - 若剩余 ≥ 6 张且接下来 6 张都接近方图（0.6 < ratio < 1.4）→ `grid6`（六宫格）
   - 若剩余 ≥ 5 张且接下来 5 张都接近方图 → `grid5`（五图）
   - 若剩余 ≥ 4 张且其中至少 3 张接近方图（0.8 < ratio < 1.25）→ `grid4`（四宫格）
   - 若剩余 = 3 → `triple`（三图版）
   - 若剩余 = 2 → `double`（双图版）
   - 每隔若干页插入 1 张 `text` 页（纯文字页，便于用户加入独白/月龄记录）
4. **版式枚举**（`PageLayoutType`，共 10 种）：`cover` · `single` · `single-portrait` · `double` · `triple` · `grid4` · `grid5` · `grid6` · `text` · `ending`
5. **降级回退**：若实际照片数量 < 版式要求（如 `grid6` 只剩 4 张），`PageView` 会自动降级到更小的版式（grid6→grid5→grid4→triple→double→single），保证不留空位

- 压缩后体积通常下降 70–90%，大幅降低 IndexedDB 占用

### 4.10 导出

| 能力 | 实现方案 | 状态 |
| --- | --- | --- |
| 导出当前页 PNG | `html2canvas` 截当前对开 DOM | ✅ |
| 导出全本长图 PNG | 循环渲染各页到离屏 DOM，拼接纵向长图 | ✅ |
| 导出多页 PDF | `exportPdf.tsx`：离屏挂载每一页 `PageView`，html-to-image (`toPng`) 优先 + html2canvas 兜底，pixelRatio=3 采样（2160×2880，≈260 DPI），`jsPDF` PNG 无损、210×280mm 自定义页面比例 | ✅ |
| 浏览器打印 | 预留 `@media print` 样式，系统打印对话框可直接出纸 | ✅ |
| 导出翻页视频 | `MediaRecorder` 录屏 或 服务端合成 | ⬜ |

**PDF 质量关键点**（v1.4）：

- 过往方案 720×960 × scale 2 ≈ 1440×1920，A4 打印仅 ~175 DPI；现按 pixelRatio=3 采样 → 2160×2880，在 210×280mm 版心上约 **260 DPI**，满足印刷级
- **优先 html-to-image**：对 `SVG mask-image`（圆形/心形照片）、`radial-gradient` / `repeating-linear-gradient`（水彩背景、胶片齿孔、赛博朋克网格）、`backdrop-filter` 等现代 CSS 支持更稳；失败时 fallback 到 `html2canvas(scale=3)`
- `jsPDF` 压缩设 `NONE` + PNG 原始字节，不破坏渐变与阴影
- PDF 页面按画册 **3:4** 自定义，不再强塞 A4 留大片白边

### 4.12 多图版式骨架变体（Layout Variants）✅

**目标**：解决"同一风格 + 同一版式永远长一个样"的审美疲劳，让用户在不改动整体风格的前提下，细粒度控制每个多图版式的排版骨架。

**数据模型**（`src/types.ts`）：

```ts
export interface LayoutVariants {
  double?: string;  // 'equal' | 'big-small' | 'stack-overlap'
  triple?: string;  // 'big-two' | 'row' | 'scatter'
  grid4?: string;   // 'grid-2x2' | 'scatter' | 'hero-right'
  grid5?: string;   // 'hero-left' | 'hero-center' | 'top1-bottom4'
  grid6?: string;   // 'grid-3x2' | 'hero-left-5' | 'mosaic'
}
export interface Template {
  // ...
  layoutVariants?: LayoutVariants;
}
```

**变体清单**（`src/layoutVariants.ts`，共 5 × 3 = 15 种）：

| 版式 | 变体 | 视觉描述 |
| --- | --- | --- |
| double | 等分并排 / 主次 7:3 / 错落重叠 | 左右等分；左大右小；方图微旋叠放 |
| triple | 一大两小 / 横向三联 / 自由散摆 | 主图 4:5 + 两张 1:1；三张等分成行；错落偏移 |
| grid4 | 2×2 方阵 / 散摆贴纸 / 左三右一 | 四张 1:1 等分；微旋手账感；左三列 + 右大图 |
| grid5 | 左主右四 / 中心焦点 / 上一下四 | 左 4:5 + 右 2×2；中央大图四角环绕；顶部横幅 + 底部四格 |
| grid6 | 3×2 方阵 / 左主右五 / 马赛克拼贴 | 六张等分照片墙；左大右五迷你；两主 + 四副 |

**设计原则**：

- 变体只决定"照片块的位置/尺寸/旋转"，**相框、气泡、caption 等风格装饰仍由 `style` 决定**，这保证同一个变体在任何风格下都和整体观感自洽
- 内置模板**不填 `layoutVariants`**，走各 style 原有的手工骨架（向后兼容，不影响 v1.0–v1.2 的视觉）
- 自定义模板**默认启用变体**，若显式填写则走通用 `DoubleVariant / TripleVariant / Grid4Variant / Grid5Variant / Grid6Variant` 渲染器
- 所有变体都使用接近方形（1:1 / 4:5）的画框，避免照片被拉变形

**编辑器体验**（`TemplateEditorPage`）：

- 右侧预览区由 3 个 tab（封面/双图/三图）扩展到 6 个 tab（封面/2/3/4/5/6 图），切到任一版式都能实时看到所选骨架
- 左侧表单增加 5 个 `VariantPicker`，每个 picker 显示 3 张基于真实骨架比例绘制的 SVG 缩略图 + 文字说明；点击缩略图立即联动预览 tab 切换

### 4.15 画册级主题（BookTheme）✅

**目标**：同一个模板，多本画册依然能"长得不一样"——允许每本画册在**模板之上**做字段级覆盖（换封面、换标题色、换相框色等），不用新建模板。

**实现**（`src/bookTheme.ts`）：

- 数据结构附加在 `Book.theme` 字段（可选）：`{ photoFrameColor?: string; titleColor?: string; coverPhotoSrc?: string; ... }`
- `PageView` 在渲染时，如检测到 `photoFrameColor` 等覆盖项则优先使用；未覆盖的字段回落到 `Template.colors`
- PDF 导出（`exportPdf.tsx`）透传同一覆盖项，保证**导出和预览视觉一致**
- 画册详情页提供"主题编辑"面板，现场调整立即预览

**与自定义模板的区别**：

- 自定义模板：一次配置，**多本画册可复用**
- 画册主题：**当前这本**的局部微调（如"这本用红色相框，下本用原色"），不影响模板本身

  | 'cover' | 'single' | 'single-portrait'
  | 'double' | 'triple'
  | 'grid4' | 'grid5' | 'grid6'
  | 'text' | 'ending';

export interface LayoutVariants {
  double?: string; triple?: string;
  grid4?: string; grid5?: string; grid6?: string;
}

export interface Template {
  id: string;
  name: string;
  category: TemplateCategory;
  style: TemplateStyle;
  description: string;
  isFree: boolean;
  colors: { bg: string; paper: string; primary: string; accent: string; text: string };
  fontFamily: { title: string; body: string };
  backgroundPattern?: string;        // CSS 底纹叠加字符串
  decorations: string[];              // emoji / 符号
  defaultTitle: string;
  defaultSubtitle: string;
  coverPhotoSrc?: string;             // 模板专属封面（可选，通常为内嵌 SVG Data URL）
  /** 仅用户自定义模板使用，内置模板走各 style 手工骨架 */
  layoutVariants?: LayoutVariants;
}

/** 画册级主题覆盖：在不改模板的前提下对单本画册做个性化 */
export interface BookThemeOverride {
  colors?: Partial<Template['colors']>;
  fontFamily?: Partial<Template['fontFamily']>;
  backgroundPattern?: string | null;  // 'none' 可清除
  photoFrameColor?: string | null;    // 覆盖所有 style 的相框/描边色
}

export type PhotoShape =
  | 'rect' | 'rounded' | 'circle' | 'heart' | 'star' | 'hexagon';

export interface BookPage {
  id: string;
  layout: PageLayoutType;
  photoIds: string[];
  title?: string; subtitle?: string; caption?: string;
  /** 多图版式的骨架变体（不填则继承 template.layoutVariants） */
  variant?: string;
  /** 每个 slot 的相框形状（与 photoIds 按索引对齐） */
  photoShapes?: (PhotoShape | undefined)[];
}

export interface Photo {
  id: string; src: string; width: number; height: number;
  ratio: number; isCover?: boolean; takenAt?: number;
}

export interface Book {
  id: string;
  title: string;
  babyName: string;
  dateRange: string;
  templateId: string;
  pages: BookPage[];
  photos: Photo[];       // 资源表：照片 dataURL + 元数据
  theme?: BookThemeOverride;
  createdAt: number;
  updatedAt: number;
}

## 十、风险与对策

| 风险 | 对策 |
| --- | --- |
| 本地存储被清理导致丢失 | 显著提示 + M3 引入云同步；自定义模板同受此影响，后续提供 JSON 导出备份 |
| 大量高清照片占用浏览器配额 | 端侧压缩已上线；后续提供"存档包"导出备份 |
| 模板视觉同质化 | 每个风格持续扩款；引入外部插画师协作；开放自定义模板让用户自行扩展 |
| 用户自定义模板质量参差、可能配出低可读性配色 | 编辑器提供实时预览 + 后续引入对比度/字体尺寸的软提示；官方推荐库仅收录内置模板 |
| 翻页动画在低端机卡顿 | 预留"简化模式"开关（关闭 3D 翻转，仅淡入淡出） |
| 版权风险（字体 / 装饰素材） | 全部使用商用可免费字体（或授权字体）；装饰自绘或 CC0；自定义模板中的 emoji 为系统字符，不涉及第三方素材 |
