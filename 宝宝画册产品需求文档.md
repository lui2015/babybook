### 1.2 名词定义

| 术语 | 说明 |
| --- | --- |
| 画册（BabyBook / Book） | 由多张宝宝照片按模板排版生成的电子相册（MVP 阶段仅 Web 端本地生成） |
| 模板（Template） | 决定画册整体风格的版式方案，包括封面、内页、背景、字体、装饰元素、配色等 |
| 内置模板（Built-in Template） | 随版本发布、以代码常量形式内置的官方模板（当前 10 款），用户不可修改 |
| 用户模板 / 自定义模板（User Template） | 用户通过可视化编辑器自行创建并保存在本地 IndexedDB 中的模板，可随时编辑和删除 |
| 风格（TemplateStyle） | 模板大类，如水彩手绘、萌趣卡通、清新极简、复古胶片、中国红、圣诞节 |
| 页面（BookPage） | 画册中的单页，由图片位、文字位、版式类型（layout）组成 |
| 版式（PageLayoutType） | 单页图文的排列组合方式，如封面、单图、三宫格、四宫格等 |
| 双页书本（Spread） | 一本画册在预览/详情页中以"对开书"形式呈现，左右两页 + 翻页动画 |

## 二、项目背景与目标

### 2.1 背景

家长在宝宝成长过程中会拍摄海量照片（月龄照、节庆照、日常照），普遍存在三类痛点：

1. **整理难**：照片散落在相册、微信、云盘，缺乏结构化归档。
2. **呈现难**：专业排版门槛高；现成第三方相册 App 普遍风格单一、节庆/宝宝向模板稀缺。
3. **分享/留念难**：希望生成可分享的链接/图片，或打印成实体画册作为成长礼物。

### 2.2 产品定位

一款面向家长的「**AI 辅助排版 + 丰富成长主题模板**」画册工具。MVP 阶段以 **Web 端本地运行** 为起点（所有数据存浏览器 IndexedDB，不上传服务器），聚焦"挑模板 → 上传照片 → 一键生成 → 预览 → 本地导出"的主链路。

### 2.3 目标用户

- **核心**：0–6 岁宝宝家长（妈妈占主导，20–45 岁）
- **扩展**：祖父母希望收到打印画册；幼师制作班级成长册

### 2.4 产品目标（MVP v1.2）

| 目标 | 衡量指标 | 状态 |
| --- | --- | --- |
| 低门槛出片：非设计师 5 分钟内完成一本画册 | 创建耗时 ≤ 5min；向导步数 ≤ 4 | 已实现（4 步向导） |
| 丰富模板：至少覆盖 5 大风格 × 10 款起步 | 模板数 ≥ 10 | 已实现（10 款 / 6 风格） |
| 开放模板体系：让用户能自己创建模板 | 可视化编辑 + 实时预览 + 本地保存 | 已实现（v1.2） |
| 智能排版：根据照片尺寸/数量自动组版 | 支持 1/2/3/4 图混合版 + 竖图专版 | 已实现（8 种版式） |
| 无需账号即可体验 | 本地 IndexedDB，刷新不丢 | 已实现 |
| 可导出：图片格式可分享 | 单页 PNG + 整本长图 PNG | 已实现 |
| 云端同步 / 账号体系 | - | 待实现 |
| PDF / 视频 / 实体印刷 | - | 待实现 |
| 模板导入/导出（JSON 分享） | - | 待实现 |

## 四、详细需求

### 4.1 首页 `/` ✅

**目标**：让新用户在 5 秒内理解产品能做什么，并给出清晰的下一步。

**页面结构**（由 `src/pages/HomePage.tsx` 实现）：

1. Hero 区：主标题 + 副标题 + 两个 CTA
   - 主 CTA：「开始创建画册」→ `/create`
   - 次 CTA：「浏览模板库」→ `/templates`
2. 特色模板展示：横排展示 3–4 款精选模板（点击进入模板预览）
3. 三大价值点图文：智能排版 / 丰富模板 / 一键分享
4. 底部入口：「我的画册」→ `/my`

### 4.2 模板库 `/templates` ✅

**目标**：让用户在进入创建流程前/中快速找到合适的模板，并管理自己创建的自定义模板。

**功能**（由 `src/pages/TemplatesPage.tsx` + `src/components/TemplatePreviewModal.tsx` 实现）：

- **筛选标签**：
  - 按 `TemplateStyle` 六大风格过滤（水彩手绘 / 萌趣卡通 / 清新极简 / 复古胶片 / 中国红 / 圣诞）
  - 按**来源**过滤：全部 / 官方模板 / 我的模板（自定义）
- **搜索**：按模板名 / 描述关键词实时过滤
- **模板卡**：
  - 封面缩略图 + 名称 + 风格标签 + 简短描述
  - 自定义模板卡片右上角显示「我的」徽标
  - 卡片底部操作：**预览**（所有模板通用）/ **编辑** · **删除**（仅自定义模板）
  - 点击卡片主体打开预览
- **顶部操作**：右上角「+ 创建自定义模板」按钮 → `/templates/new`
- **预览弹层**：
  - 全屏弹出，展示该模板的**示例画册**（使用内置示例照片）
  - 以**双页书本对开**形式呈现，带翻页动画
  - 左右半区点击 / `←` `→` 键盘翻页 / 页码圆点跳转
  - 底部紧凑控件条：翻页按钮 + 页码圆点 + 页码文本
  - **关键体验**：BookStage 组件通过 `ResizeObserver` 实测可用宽高，按 3:2 双页比例夹紧，**保证任何窗口尺寸下都完整展示整本画册，不需要滚动**
  - 右上角「使用此模板」→ 跳转 `/create`（携带 `templateId`）

### 4.3 创建画册 `/create` ✅

**目标**：一条通畅主路径，引导用户 5 分钟内出片。

**流程**（由 `src/pages/CreatePage.tsx` 实现，共 4 步）：

#### 步骤 1：上传照片
- 支持多选上传（图片格式 jpg/png/webp）
- 上传后**立即在浏览器内压缩**（`src/imageUtils.ts`）：
  - 最长边 ≤ 1600px
  - JPEG 质量 0.85
  - 并发度 3（避免主线程卡顿）
- 缩略图网格展示，支持删除、调整顺序（拖拽）
- 最少 4 张，最多 60 张（超出提示）

#### 步骤 2：基本信息
- 画册标题（必填，默认「XX 的成长画册」）
- 宝宝昵称、日期范围（可选）
- 封面主标题、副标题（可选，默认由模板生成）

#### 步骤 3：选择模板
- 以卡片网格展示全部模板（内置 10 款 + 用户自定义模板）
- 点击卡片高亮选中，可点「预览」打开双页对开预览弹层
- 也支持从模板库 `/templates` 深链带入 `templateId`（含自定义模板 id）

#### 步骤 4：生成并预览
- 进度条动画（压缩 → 排版 → 渲染）
- 排版引擎（`src/layoutEngine.ts`）根据照片尺寸、数量智能选版式（详见 4.6）
- 完成后自动跳转 `/book/:id`

### 4.4 我的画册 `/my` ✅

**目标**：快速回到任意一本已创建的画册。

**功能**（由 `src/pages/MyBooksPage.tsx` 实现）：

- 卡片网格展示：封面缩略 + 标题 + 页数 + 创建时间（相对时间）
- 点击卡片 → 进入 `/book/:id`
- 右上角删除按钮（二次确认后从 IndexedDB 删除）
- 空态：插画 + 「创建第一本画册」CTA

### 4.5 画册详情 `/book/:id` ✅

**目标**：让画册以"真实书本"的形式被欣赏和分享。

**功能**（由 `src/pages/BookDetailPage.tsx` 实现）：

- **双页书本预览**：左页 + 右页对开，封面单页居中
- **翻页动画**：CSS 3D transform，约 600ms 缓动翻页
- **交互**：
  - 点击左右半区翻页
  - `←` / `→` 键盘翻页
  - 底部页码圆点跳转到任意对开
- **导出**：
  - 「导出当前页 PNG」：`html2canvas` 截取当前对开
  - 「导出长图 PNG」：全书所有页纵向拼接为一张长图
- **编辑**（轻量）：修改画册标题、切换模板（重新排版）

### 4.6 智能排版引擎 ✅

**目标**：让照片自动"落位"到最合适的版式，减少用户手动调整。

**规则**（由 `src/layoutEngine.ts` 实现）：

1. **首页**：始终为 `cover`（封面）
2. **末页**：始终为 `ending`（结束页）
3. **内页（根据剩余照片队列动态取用）**：
   - 若下一张照片 **ratio ≥ 1.25**（明显横图）→ `single`（整页单图横幅）
   - 若下一张照片 **ratio ≤ 0.8**（明显竖图）→ `single-portrait`（竖图专版，右侧留白放文字）
   - 若剩余 ≥ 4 张且其中至少 3 张接近方图（0.8 < ratio < 1.25）→ `grid4`（四宫格）
   - 若剩余 = 3 → `triple`（三图版）
   - 若剩余 = 2 → `double`（双图版）
   - 每隔若干页插入 1 张 `text` 页（纯文字页，便于用户加入独白/月龄记录）
4. **版式枚举**（`PageLayoutType`）：`cover` · `single` · `single-portrait` · `double` · `triple` · `grid4` · `text` · `ending`

### 4.7 模板体系 ✅

**模板体系分为两层**：

1. **内置模板**（Built-in）：以代码常量 `TEMPLATES` 形式随版本发布，当前 10 款，用户不可修改；
2. **用户模板**（User / Custom）：用户通过编辑器自行创建，存于 IndexedDB（`src/userTemplates.ts`），id 前缀 `tpl_user_`；
3. 两层在运行时由 `TemplateRegistry`（`src/TemplateRegistry.tsx`）合并，对外暴露统一的 `allTemplates / getTemplate / refresh` API，**所有消费方（创建向导、模板库、画册详情、我的画册）读取到的都是合并后的列表**，对自定义模板完全透明。

**6 大风格（`TemplateStyle`）**：

| 风格 | 代码值 | 视觉语言 |
| --- | --- | --- |
| 水彩手绘 | `watercolor` | 手写斜贴标签 + 柔和圆角相框 |
| 萌趣卡通 | `cartoon` | 气泡对话框 + 圆角糖果边 |
| 清新极简 | `minimal` | 杂志式排版 + 大留白 + 细分割线 |
| 复古胶片 | `vintage` | Polaroid 白边 + 打字机体 + 日期印章 |
| 中国红 | `festival-cn` | 红边框 + 竖排标题 + 印章（年节场景） |
| 圣诞 | `festival-xmas` | 雪花边 + 松枝丝带（冬/圣诞场景） |

**当前内置 10 款模板**（`src/templates.ts`），每款模板定义：

- `id` / `name` / `description`
- `style`（上述 6 类之一）
- `colors`：背景色 / 主色 / 辅色 / 文字色
- `fonts`：中文字体 / 英文字体
- `decorations`：装饰元素 URL 或 SVG 路径

用户自定义模板复用同一 `Template` 接口，因此渲染层（`PageView`）、排版引擎（`generatePages`）、预览弹层（`TemplatePreviewModal`）**零改动**即可适配。

后续扩展计划：每个风格维度持续扩充到 ≥ 10 款（目标总数 60+）；同时为用户模板提供 JSON 导入/导出以便分享。

### 4.8 本地存储 ✅

- 使用 `idb-keyval` 封装 IndexedDB（`src/storage.ts`）
- 存储内容：
  - 画册列表（`books` 键，数组）
  - 每本画册包含：`id` · `title` · `templateId` · `pages[]` · `createdAt` · `updatedAt` · `coverThumbnail`
  - 照片以 base64 data URL 形式内嵌在 `BookPage.photos[]`
- **不上传任何数据到服务器**，满足隐私合规默认态；换浏览器/清缓存后数据丢失（风险在首页底部通过小字提示）

### 4.9 端侧图片压缩 ✅

- Canvas 重绘 + `toBlob('image/jpeg', 0.85)`
- 压缩并发：Promise 池，同时处理 3 张，避免主线程冻结
- 压缩后体积通常下降 70–90%，大幅降低 IndexedDB 占用

### 4.10 导出

| 能力 | 实现方案 | 状态 |
| --- | --- | --- |
| 导出当前页 PNG | `html2canvas` 截当前对开 DOM | ✅ |
| 导出全本长图 PNG | 循环渲染各页到离屏 DOM，拼接纵向长图 | ✅ |
| 导出 PDF | `jsPDF` + 分页渲染（印刷级 300dpi） | ⬜ |
| 导出翻页视频 | `MediaRecorder` 录屏 或 服务端合成 | ⬜ |

### 4.11 自定义模板 ✅

**目标**：打破"只能用官方 10 款"的天花板，让用户能根据自己的主题、配色偏好、节庆场景自行定义模板，且所有定制体现在**真实的 PageView 渲染里**，做到"所见即所得"。

**入口**：
- 模板库 `/templates` 右上角「+ 创建自定义模板」→ `/templates/new`
- 自定义模板卡片底部「编辑」按钮 → `/templates/edit/:id`
- 自定义模板卡片底部「删除」按钮（二次确认）

**编辑器页面**（由 `src/pages/TemplateEditorPage.tsx` 实现，左侧表单 / 右侧实时预览）：

| 配置分组 | 可编辑字段 | 控件 |
| --- | --- | --- |
| 基础信息 | 名称、描述、分类（复用 5 类）、是否标记免费 | 文本框 + 下拉 + 复选 |
| 视觉风格骨架 | `style` = watercolor / cartoon / minimal / vintage / festival-cn / festival-xmas | 6 个按钮单选（复用现有 `PageView` 风格分支） |
| 配色方案 | 5 个主题色：`bg` / `paper` / `primary` / `accent` / `text` | 颜色选择器 + 手输 hex |
| 字体 | 标题字体 / 正文字体 | 下拉（手写 / 衬线 / 打字机 / 楷体 / 宋体 / 黑体 …） |
| 背景底纹 | 6 种预设（柔光圆斑 / 斜纹 / 双渐变 / 格纹 / 双色圆点 / 纯色）一键应用，或直接编辑 CSS `background` 字符串 | 预设卡片 + textarea |
| 装饰元素 | emoji / 符号列表（逗号分隔，最多 6 个） | 单行输入 |
| 默认文案 | 默认封面标题 / 副标题 | 文本框 |

**实时预览**：右侧渲染 3 页样板（封面 / 双图 / 三图），直接调用真实 `PageView` 组件并传入当前编辑中的模板对象，改任何字段右侧即时刷新——所见即所得。

**存储**：

- 专用 IndexedDB 存储层 `src/userTemplates.ts`，key 前缀 `babybook:tpl:`，与画册数据隔离
- 自定义模板 id 统一前缀 `tpl_user_`，与内置模板（`tpl_*`）区分
- 提供 `list / save / get / delete` API

**全局注册表**（`src/TemplateRegistry.tsx`）：

- React Context 在 App 挂载时加载一次用户模板列表，之后通过 `refresh()` 增量刷新
- 对外统一暴露 `allTemplates: Template[]`（内置 + 用户）与 `getTemplate(id): Template | undefined`
- 让其它页面（`CreatePage` / `BookDetailPage` / `MyBooksPage`）对模板来源无感

**兜底与安全**：

- 用户删除某个自定义模板后，仍在使用该模板的历史画册在 `BookDetailPage` 上显示「模板不存在，请重新选择模板」的兜底提示，不影响打开
- 画册内仅保存 `templateId` 字符串，模板样式始终由注册表动态解析，避免历史数据与当前模板定义脱节
- 自定义模板不上传任何服务器，完全本地存储（与画册数据一致）

**后续增强（待实现）**：

- JSON 导入/导出（模板分享）
- 细粒度版式级样式覆盖（每个 `PageLayoutType` 单独微调）
- 装饰元素素材库（从 SVG/贴纸库选择，而非仅 emoji）
- 模板复制：从任意现有模板"另存为"一份作为自定义模板的起点

## 十、风险与对策

| 风险 | 对策 |
| --- | --- |
| 本地存储被清理导致丢失 | 显著提示 + M3 引入云同步；自定义模板同受此影响，后续提供 JSON 导出备份 |
| 大量高清照片占用浏览器配额 | 端侧压缩已上线；后续提供"存档包"导出备份 |
| 模板视觉同质化 | 每个风格持续扩款；引入外部插画师协作；开放自定义模板让用户自行扩展 |
| 用户自定义模板质量参差、可能配出低可读性配色 | 编辑器提供实时预览 + 后续引入对比度/字体尺寸的软提示；官方推荐库仅收录内置模板 |
| 翻页动画在低端机卡顿 | 预留"简化模式"开关（关闭 3D 翻转，仅淡入淡出） |
| 版权风险（字体 / 装饰素材） | 全部使用商用可免费字体（或授权字体）；装饰自绘或 CC0；自定义模板中的 emoji 为系统字符，不涉及第三方素材 |
